Skill 构建完全指南

技能（Skill）是一组指令的集合——以简单的文件夹形式封装——它教会 OpenClaw 如何处理特定的任务或工作流。

第一章：基础

📁 技能文件结构

your-skill-name/
├── SKILL.md              # 必需 - 主要技能文件
├── scripts/              # 可选 - 可执行代码
│   ├── process_data.py
│   └── validate.sh
├── references/           # 可选 - 文档
│   ├── api-guide.md
│   └── examples/
└── assets/              # 可选 - 模板等
    └── report-template.md

🎯 核心设计原则

  • 渐进式披露 - 三层系统：YAML frontmatter → SKILL.md 正文 → 附加文件
  • 可组合性 - 技能应与其他技能良好配合
  • 可移植性 - 一次创建，可在所有平台使用

📝 YAML Frontmatter（最重要）

---
name: your-skill-name
description: 它做什么。当用户要求[特定短语]时使用。
---

第二章：规划与设计

💡 从用例开始

示例：项目冲刺规划

  • 触发：用户说"帮我规划这个冲刺"
  • 步骤：获取项目状态 → 分析团队容量 → 建议优先级 → 创建任务
  • 结果：完整规划好的冲刺

⚠️ 关键规则

  • SKILL.md 必须精确命名（区分大小写）
  • 文件夹使用 kebab-case（不要空格、下划线、大写）
  • description 必须包含：技能功能 + 触发条件
  • 描述不超过 1024 字符

第三章：测试与迭代

✅ 推荐测试方法

1. 触发测试

  • ✅ 在明显任务上触发
  • ✅ 在改写请求上触发
  • ❌ 不在不相关主题上触发

2. 功能测试

  • 生成有效输出
  • API 调用成功
  • 错误处理有效

3. 性能比较

  • 无技能 vs 有技能
  • 对比 token 消耗
  • 对比成功率

第四章：分发与共享

🚀 推荐方法

  • 托管在 GitHub - 开源技能的公共仓库
  • 在 MCP 仓库中文档化 - 从 MCP 文档链接到技能
  • 创建安装指南 - 清晰的上手说明

第五章：模式与故障排除

🔧 常见模式

模式 1：顺序工作流编排

当用户需要特定顺序的多步骤流程时使用

模式 2：多 MCP 协调

当工作流跨越多个服务时使用

模式 3：迭代改进

当输出质量随迭代提高时使用

模式 4：领域特定智能

当技能添加超越工具访问的专业知识时使用

❌ 故障排除

技能不上传

  • 错误 "Could not find SKILL.md" → 重命名为 SKILL.md
  • 错误 "Invalid frontmatter" → 检查 YAML 格式
  • 错误 "Invalid skill name" → 使用 kebab-case
